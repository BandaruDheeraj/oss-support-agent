## 2) What competitors do (why easier)

- **Competitor mechanism:** Braintrust row spans can log `input`, `output`, and `scores` directly.
- **Why easier:** Evaluation data model is first-class in the SDK, reducing schema work in app code.

## 3) What we do today in oss-support-agent

- **Current behavior:** Run summary and per-issue scores are emitted through manual OpenInference attributes per adapter.
- **Affected components:** evals/platforms/arize.ts, evals/platforms/braintrust.ts

## 4) Gap statement

Because Run summary and per-issue scores are emitted through manual OpenInference attributes per adapter., users cannot Log eval rows and score summaries without hand-authored span schemas as easily as with Braintrust row spans can log `input`, `output`, and `scores` directly..

## 5) Proposed fix

- **Owner-scoped change:** Add `logEvaluationRow` and `logEvaluationSummary` APIs in Arize JS SDK with defaults for score keys.
- **Root cause:** No first-class eval-row/eval-summary helper in Arize JS path.
