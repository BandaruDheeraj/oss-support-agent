## 2) What competitors do (why easier)

- **Competitor mechanism:** LangSmith offers direct dataset creation in setup (`createDataset`).
- **Why easier:** A single SDK path handles dataset provisioning as part of bootstrap.

## 3) What we do today in oss-support-agent

- **Current behavior:** Arize adapter skips cloud dataset provisioning and records a setup note about dashboard/tabular API dependency.
- **Affected components:** evals/platforms/arize.ts, evals/platforms/langsmith.ts

## 4) Gap statement

Because Arize adapter skips cloud dataset provisioning and records a setup note about dashboard/tabular API dependency., users cannot Create datasets consistently for local and cloud setups as easily as with LangSmith offers direct dataset creation in setup (`createDataset`)..

## 5) Proposed fix

- **Owner-scoped change:** Add `createDatasetIfMissing` for Phoenix local and Arize cloud with one contract.
- **Root cause:** No symmetric local/cloud dataset creation helper in Arize JS tooling.
